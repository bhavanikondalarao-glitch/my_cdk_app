# ğŸ“˜ **RollingUpdate Strategy â€” Definition**

**RollingUpdate is a Kubernetes Deployment strategy where old pods are gradually replaced by new pods in small batches.
This ensures zero-downtime because some pods stay running while others are updated.**

Two main settings control the speed and availability:

* **maxUnavailable** â†’ How many pods can be unavailable (deleted) at the same time
* **maxSurge** â†’ How many extra pods (above the desired count) Kubernetes can create during update
